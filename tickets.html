Ticket System

Efficient support and moderation through organized tickets

Ticket Rules & Purpose

Our ticket system is designed to provide structured support and maintain order in your server. Here's what you need to know:

Purpose

Tickets are meant for specific support requests, moderation issues, or private discussions that shouldn't be handled in public channels.

Proper Use

Create tickets only when you genuinely need staff assistance. Common uses include reporting rule violations, asking sensitive questions, or requesting special permissions.

Improper Use

Don't create tickets for general chat, random questions that could be asked in public channels, or to test if the system works.

Goal

The ticket system aims to provide a private, organized way to handle issues that require staff attention while keeping your main channels clean and on-topic.

Ticket Etiquette & Management

Proper ticket management ensures efficient support for everyone in your server.

For Server Members

  • Be patient - staff members may be handling multiple tickets
  • Provide all relevant information upfront to avoid back-and-forth
  • Don't ping staff repeatedly - they'll respond when available
  • Keep all communication in the ticket channel

For Server Staff

  • Respond to tickets in a timely manner
  • Clearly communicate expected resolution times
  • Use ticket categories to organize different request types
  • Close tickets only when the issue is fully resolved

Best Practices

Set clear expectations in your server rules about what warrants a ticket and how long members should expect to wait for responses based on your team's availability.

Ticket Content Policy

To maintain security and privacy, please adhere to these content guidelines:

Sensitive Information

Never share passwords, API keys, or other sensitive credentials through tickets. Staff will never ask for this information.

NSFW Content

Tickets should not be used to store or share NSFW material. Use appropriate channels for such content if allowed by your server rules.

Privacy Protection

Don't share personal information about yourself or others that could compromise privacy or security.

Consequences

Violating these policies may result in ticket closure, warnings, or other moderation actions based on your server's rules.

Setting Up Tickets

Configure the ticket system in your server with these simple steps:

Basic Setup

Use the command /setupticket to initialize the ticket system in your server. This creates the necessary channels and roles.

Custom Rules

Optionally include rules with the setup command: /setupticket [rules]. These will be displayed when members create tickets.

Default Rules

If you don't specify custom rules, the system will use our default ticket rules.

Advanced Configuration

After setup, use /ticketsettings to customize:

  • Ticket categories
  • Staff roles
  • Auto-close timers
  • Transcript options

Pro Tip:You should set up the ticket system as soon as possible to ensure a smooth experience for your members.
